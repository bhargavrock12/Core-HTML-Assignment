Text Formatting Tags


  1. <b> (Bold)

    • Use Case: Makes text bold without adding any extra importance.
    • Best Practice: Use <strong> for indicating importance or emphasis, as it is more semantically meaningful.

  2. <strong> (Strong Emphasis)

    • Use Case: Highlights text that is of strong importance or urgency.
    • Best Practice: Preferred over <b> for text that needs emphasis, as it is both visually bold and semantically meaningful.

  3. <i> (Italic)

    • Use Case: Italicizes text without adding any extra emphasis.
    • Best Practice: Use for stylistic purposes, such as denoting a book title or foreign phrase. Use <em> when indicating emphasis.

  4. <em> (Emphasis)

    • Use Case: Emphasizes text, typically by italicizing, and also provides semantic meaning.
    • Best Practice: Preferred over <i> for emphasizing words, as it conveys meaning to screen readers.

  5. <u> (Underline)

    • Use Case: Underlines text.
    • Best Practice: Avoid using <u> for underlining, as it can be confused with links. Use CSS for custom styling instead.

  6. <mark> (Highlight)

    • Use Case: Highlights or marks text with a yellow background, indicating that it's relevant or important.
    • Best Practice: Use sparingly for important information that needs to stand out, such as search results.

  7. <small> (Small Text)

    • Use Case: Reduces the size of the text.
    • Best Practice: Use for disclaimers, footnotes, or secondary information. Avoid overusing as it can reduce readability.

  8. <sub> (Subscript)

    • Use Case: Displays text as subscript, often used in chemical formulas or mathematical expressions.
    • Best Practice: Use for subscripted text in formulas or equations, such as H<sub>2</sub>O, which gives H2O.

  9. <sup> (Superscript)

    • Use Case: Displays text as superscript, often used in footnotes or mathematical expressions.
    • Best Practice: Use for superscripted text, such as exponents (e.g., x<sup>2</sup> ,which gives x2).

  10. <ins> (Inserted Text)

    • Use Case: Indicates inserted text, typically underlined by default.
    • Best Practice: Use for showing content additions, such as in track changes or document edits.

  11. <del> (Deleted Text)

    • Use Case: Indicates text that has been deleted, typically strikethrough by default.
    • Best Practice: Use for showing content removals or corrections.

  12. <abbr> (abbreviation)

    • Use Case: Defines an abbreviation or acronym, which provides the full term on hover.
    • Best Practice: Always provide a 'title' attribute to explain the abbreviation. For example:<abbr title="Hypertext Markup Language">HTML</abbr> ,which gives HTML.

  13. <code> (inline code)

    • Use Case: Displays text in a monospaced font, typically for code snippets.
    • Best Practice: Use for inline code examples within a block of text. Combine with <pre> for block-level code.

  14. <pre> (Preformatted Text)

    • Use Case: Preserves whitespace and line breaks, displaying text exactly as written.
    • Best Practice: Use for multi-line code snippets or preformatted text, such as ASCII art.

  15. <blockquote> (Block Quotation)

    • Use Case: Indicates a long quotation, usually from another source.
    • Best Practice: Use for quoting long passages. Use <q> for shorter, inline quotes.

  16. <q> (Inline Quotation)

    • Use Case: Represents a short, inline quotation.
    • Best Practice: Use for shorter quotes within a paragraph. It automatically adds quotation marks.
      eg., Use <q></q>, it gives When you share your secrets with wind, you can't blame wind for sharing it with tree

  17. <cite> (Citation)

    • Use Case: Represents a reference to a creative work, such as a book, article, or website.
    • Best Practice: Use for citing the title of a work, not for quoting content.
      eg., I recently read To Kill a Mockingbird by Harper Lee.

  18. <kbd> (Keyboard Input)

    • Use Case: Displays text as keyboard input, usually in a monospaced font.
    • Best Practice: Use for representing keyboard commands, such as <kbd>Ctrl</kbd> + <kbd>C</kbd>,gives Ctrl + C.

  19. <var> (Variable)

    • Use Case: Represents a variable in a mathematical expression or programming context.
    • Best Practice: Use when indicating variables within code or equations.

  20. <s> (Strikethrough)

    • Use Case: Strikes through text, typically to indicate something that is no longer relevant or has been changed.
    • Best Practice: Use sparingly; <del> may be more appropriate for indicating deletions.

  21. <bdi> (Bi-directional Isolation)

    • Use Case: Isolates a span of text that may be formatted in a different direction (e.g., right-to-left languages).
    • Best Practice: Use when embedding text that could disrupt the text direction of the surrounding content.

  22. <bdo> (Bi-directional Override)

    • Use Case: Overrides the current text direction to display in a different direction.
    • Best Practice: Use for cases where text direction must be explicitly set, such as for RTL languages in a LTR context.

  23. <time> (Time)

    • Use Case: Represents a specific time or date.
    • Best Practice: Use with a datetime attribute for machine-readable formats. For example: August 11, 2024.
